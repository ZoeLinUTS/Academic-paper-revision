# 用户手册

如何使用 `academic-paper-revision` skill、给它什么、它返回什么，以及两种运行模式的区别。

> [English](../USER-GUIDE.md) | **中文** | [日本語](USER-GUIDE.ja.md) | [한국어](USER-GUIDE.ko.md) | [Español](USER-GUIDE.es.md) | [Français](USER-GUIDE.fr.md) | [Deutsch](USER-GUIDE.de.md) | [Português](USER-GUIDE.pt.md)

## 1. 开始前：需要准备什么

| 模式 | 你提供的输入 | Claude 做什么 |
| --- | --- | --- |
| **完整修订** | **LaTeX 源文件**：主 `.tex`、`references.bib`、会议的类/样式文件（`llncs.cls` + `*.bst`，或 `acl.sty`……）以及 `figures/`。一个 Overleaf **项目导出（zip）**最理想。 | 对源文件做外科手术式编辑，抵消增加的篇幅，交回重新打包的上传 zip。 |
| **仅审阅** | 一份 **PDF**（或源文件）即可。 | 只读取和评论——**不改动任何文件**。 |

完整修订模式需要源文件，因为你无法对 PDF 做外科手术式的编辑，而从 PDF 重建源文件
会损坏你的排版和数字。已渲染的 `main.pdf` 是有用的*次要*输入，可用于核对渲染时属性
（页数、浮动体位置），但它本身永远不够。

## 2. 告诉 Claude 你的目标会议

这是承重的一步。同一处编辑，对一个会议是正确的，对另一个可能就直接被拒。明确说出
会议名（ACL、NAACL、EMNLP、AJCAI、NeurIPS……）。若不清楚该会议的确切规则，Claude
会在编辑前查阅当前的征稿启事。如果文件名写的是一个会议而你的笔记写的是另一个，它会
指出这个矛盾并询问——它绝不臆测。

## 3. 选一种模式并提问

**完整修订** —— 示例提示：
- 「把这个改投 EMNLP，让它读起来像被录用的论文。」
- 「把这个修剪到 AJCAI 的页数限制，并去掉 RQ 脚手架。」
- 「为方法加一个公式/算法，并修好交叉引用。」

**仅审阅** —— 示例提示：
- 「只审阅这份草稿——给我总览和评论，什么都别改。」
- 「评估新颖性，并告诉我可能的审稿人反对意见。」
- 「这里的范围和主要问题是什么？它与先前工作有何不同？」

### 沟通语言 —— 任意语言

你可以用**任意语言**工作并以该语言获得结果。Claude 会询问（或推断）你想用哪种语言
进行*对话*——中文、西班牙语、阿拉伯语或其他任何语言——并以该语言交付每一份总览、
评论、发现和摘要。这与稿件无关：**论文仍保持英语**（或会议要求的语言），只有对话
跟随你选择的语言。

## 4. 输出长什么样

两种模式都以同一份审稿人风格的备忘开头：

1. **定位简报** —— *草稿是什么* · *核心贡献* · *新颖性（坦诚，附审稿人攻击要点）* ·
   *完成状态*。结尾确认会议并询问你想要的范围。
2. **发现清单** —— 每个问题一行，分组并按严重程度排序，每行点明修复方式。示例行：
   - `[venue]` 目标 = ACL (ARR)：正文 8 页，Limitations/Ethics 免费，双盲。
   - `[structure]` 显式的 RQ1–RQ3 脚手架 → 改为散文式贡献。
   - `[formalization]` 评分规则很核心却零公式 → 加 2–3 个公式。
   - `[rigor]` 头条提升无显著性检验 / 单一划分 → 标为头号弱点。
   - `[bug]` `\ref{fig:pipeline}` 指向无 `\label` 的图 → 补上 label。
   - `[cross-ref]` 相关工作是按时间排的清单 → 按主题重组。
   - `[page-fit]` 超出 0.5 页 → 与抵消它的具体 `[trim]` 动作配对
     （合并冗余段落、把演示例子移到附录）。

**仅审阅到此为止**（外加具体建议和一条坦诚的告诫）。不触碰任何文件。

**完整修订继续：**

3. **变更摘要**（编辑后）—— 改了什么、*为什么*，按顺序：应用的会议惯例 · 框架/结构 ·
   增加的形式化 · 图 · 构建/交叉引用修复 · 页数适配。然后是**你必须做的事**：重新编译
   （例如在 Overleaf 上）、核对指名的渲染时项目（页数、某个具体字形/浮动体），以及若某项
   核对失败时下一个可用的杠杆。它会标出任何无法在本地核实的内容和任何未确认的会议规则。
4. **一条坦诚的告诫** —— 直白说出最大的那一个审稿人风险。

## 4b. 一个实操示例（输入 → 输出）

具体来说，在附带的 `examples/student-draft/` 上运行长这样。

**期望输入** —— LaTeX 源文件（不是 PDF），外加一个指定的会议：

```
examples/student-draft/
  main.tex          # article 类草稿：RQ1–RQ3、通用标题、
                    #   零公式、\ref{fig:pipeline} 指向无 label 的图
  references.bib    # 提供了但从未 \cite
```

> 提示：*「按 **ACL (ARR)** 审阅这个，然后改写得像被录用的论文。」*

**示例输出 —— Claude 首先返回的简报 + 发现清单：**

```
定位简报
  它是什么   : 把标点保留为 token 的情感分类器；3 个章节。
  贡献       : 标点感知的分词提升电影评论上的准确率。
  新颖性     : 目前写得偏薄——想法可信但读起来像一次消融。
               审稿人攻击：「这只是预处理小改动，分析在哪？」
  完成度     : 有结果但无显著性检验；一张图渲染成 "??"。

发现（按严重程度排序）
  [venue]         目标 = ACL (ARR)：正文 8 页，Limitations/Ethics 免费，双盲。
  [structure]     显式的 RQ1–RQ3 脚手架 → 改为散文式贡献。
  [formalization] 评分规则很核心却零公式 → 加 2–3 个公式。
  [rigor]         头条提升无显著性检验 / 单一划分 → 头号弱点。
  [bug]           \ref{fig:pipeline} 指向无 \label 的图 → 补上 label。
  [cross-ref]     相关工作是按时间排的清单 → 按主题重组。
  [venue]         缺 Limitations 章节——ACL 要求且不占页 → 补上。
```

**仅审阅**到此为止（外加几条具体建议和一条坦诚的告诫）。**完整修订**接着做出编辑
并返回一份变更摘要，准确告诉你要重新编译和核对什么。可用示例的
`EXPECTED-FINDINGS.md` 给任何一次运行打分。

## 5. Overleaf 往返（完整模式）

1. 在 Overleaf：**Menu → Download → Source**（一个 zip）。
2. 把 zip（或解包后的源文件）给 Claude。
3. Claude 编辑源文件并重新打包一个只含构建输入（`.tex`、`.bib`、类/样式、`figures/`）
   的上传 zip——它排除 `main.pdf`、README 和 git 文件。
4. 把 zip 重新上传到 Overleaf 并重新编译。
5. 核对 Claude 指名的渲染时项目（页数是渲染时属性，只能在编译后确认）。

## 6. 研究诚信保证

- **绝不编造数字。** 结果占位符只从你提供的真实、已保存的运行中填入。若结果尚不存在，
  它们就保持为占位符。
- **保留你的技术内容。** 编辑是外科手术式的；智力内容保留，只改变呈现方式。
- **不擅自做共享性操作。** 未经你要求，它不会 commit、push 或发送任何东西。

## 7. 在示例上试试

附带两份合成草稿（虚构作者、数据集、数字——无 IP），每份都带一个可用来打分的
`EXPECTED-FINDINGS.md` 参考答案：

- **`examples/student-draft/main.tex`** —— *低于*页数限制，预置了经典问题：
  RQ1–RQ3 脚手架、通用标题（"Approach"、"Experiment 1"）、按时间排的相关工作章节、
  零公式（尽管评分规则很核心），以及一个 `\ref{fig:pipeline}` 但其图无 `\label`
  （渲染成 `??`）。还缺显著性检验——即 `[rigor]` 发现。
- **`examples/over-length/main.tex`** —— 目标是 ACL 短文（4 页）限制，但跑到约 5–6 页。
  它其他方面已较打磨，因此用于练习**删减步骤**：合并两个冗余的引言段落、收紧重复的
  相关工作、把一个长的演示例子移到附录。

在任一草稿上运行任一模式，就能看到简报、发现清单——以及在完整模式下——具体编辑。

## 8. 常见问题

**我只能给它 PDF 吗？** 仅审阅可以。完整修订不行——问 Claude，它会索要 LaTeX 源文件。

**它会把我的论文翻译成我的聊天语言吗？** 不会。稿件保持会议的语言；只有对话跟随你
选择的语言。

**它适用于非 ACL 会议吗？** 适用。`references/venue-conventions.md` 覆盖 ACL 系、
Springer/LNCS/AJCAI 和 ML 会议；其他任何会议 Claude 会先读当前 CFP 并记下所学。

## 9. 用于其他 AI 模型（Codex、DeepSeek、Gemini……）

此 skill 只是**与模型无关的 Markdown 指令**——只有 `SKILL.md` 顶部的 YAML front-matter
（`---` 之间的 `name:`/`description:` 块）是 Claude 专用的自动加载元数据。它下面的全部
内容都是任何有能力的模型都能遵循的纯提示。要在别处使用：

1. **复制 `SKILL.md` 的正文**——结尾 `---` *之后*的全部内容——粘贴进另一工具的
   **系统提示**（或作为第一条消息粘贴）。可以丢掉 YAML front-matter；它只对 Claude 的
   skill 自动发现有意义。
2. **附上参考文件** `references/venue-conventions.md` 作为上下文（粘贴，或加到该工具的
   文件/上下文面板），让模型拥有会议规则。
3. **以同样方式给它论文**——完整修订给 `.tex` 源文件，仅审阅给 PDF——并说明目标会议。

各工具备注：

- **OpenAI Codex / ChatGPT、DeepSeek、Gemini、本地模型：** 把 `SKILL.md` 正文粘贴为
  系统/开发者提示，把 venue-conventions 文件作为上下文，然后照上面进行。无需改动格式——
  全是散文式指令。
- **Cursor / Windsurf / Continue 及类似的 IDE 智能体：** 把 `SKILL.md` 正文放进该工具
  会读取的项目规则文件（如 `.cursorrules`、`AGENTS.md`）以便自动加载，并把 `references/`
  保留在仓库里。
- **可能不同之处：** 工具调用与文件编辑通过各宿主自身的机制发生，因此没有仓库写权限的
  模型仍可做**仅审阅**（只读取和评论），但**完整修订**需要一个具备编辑能力的宿主。
  研究诚信规则——*绝不编造结果数字*——写在提示本身里，因此它会传递给任何遵循这些指令
  的模型。
